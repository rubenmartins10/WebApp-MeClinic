## 📝 Como Contribuir para a Documentação

Se encontrar erros ou quiser adicionar informações:

1. Edite o arquivo correspondente
2. Mantenha o mesmo formato Markdown
3. Use emojis para visuall clarity
4. Teste links internos
5. Atualize esta página (INDEX.md) se trocar estrutura

### Padrão de Escrita

```markdown
# 🎯 Seção Principal

## 📋 Subseção

**Negrito:** para conceitos
`código inline` para referências de código
### Código em bloco para exemplos

- Listas com bullets
- São boas para visão geral

| Tabelas | São | Ótimas |
|---------|-----|--------|
| Para    | Ref.| rapida |
```
